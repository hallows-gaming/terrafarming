# TerraFarming UI改善バックログ

この文書は、今後検討するUI改善候補を管理するための一覧です。

## 運用ルール

- 関連する機能を更新するときは、この一覧を確認する。
- 同時に実施すると効率がよい項目があれば、実装前にユーザーへ提案する。
- 提案時は「今回一緒に行う項目」と「後回しにする項目」を明示する。
- 未確認の項目を、ついでという理由だけで自動実装しない。
- 完了した項目はチェックを付け、仕様変更があれば本文も更新する。

## 優先度A: 画面構造の基盤

- [x] 下部ナビを常時固定し、コンテンツ部分だけをスクロールさせる。
- [x] スマートフォンでは画面全体を使い、PCでは390pxから430px程度の表示幅を維持する。
- [x] `safe-area-inset-bottom` に対応する。
- [x] タブ内の階層画面に固定ヘッダーと戻るボタンを設ける。
- [x] 戻ったときに一覧のスクロール位置と選択状態を維持する。
- [ ] 画面階層は原則「下部タブ、機能画面、詳細モーダル」までとする。
- [ ] モーダルを重ねて開かない。
- [ ] タップ領域を原則44px以上にする。
- [ ] 押せる要素と情報表示だけの要素を視覚的に区別する。
- [ ] 処理中の二重操作を防止する。

## 優先度A: HUD整理

- [ ] 大きな環境HUDは畑タブだけに表示する。
- [ ] 畑以外では、その画面で必要な通貨や件数だけを小さく表示する。
- [ ] 畑上部の惑星ビューを100pxから130px程度の横長表示に整理する。
- [ ] 惑星ビューに開拓度と開拓段階を重ねて表示する。
- [ ] 惑星ビューを押すと環境詳細画面を開く。
- [ ] コイン、食料、酸素、水分、窒素などを将来イラストアイコンへ差し替える。
- [ ] 同じ情報を複数箇所へ重複表示しない。
